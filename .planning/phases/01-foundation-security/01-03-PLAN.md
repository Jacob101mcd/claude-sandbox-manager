<task type="auto">
  <name>Task 1: Create lib/ssh.sh for SSH key lifecycle management</name>
  <files>lib/ssh.sh</files>
  <action>
Create `lib/ssh.sh` with the following functions, adapting from PowerShell `Ensure-SshKeys`, `Stage-SshKeys`, and `Write-SshConfig` in scripts/common.ps1:

1. `ssh_ensure_keys(name)`:
   - Get SSH dir via `common_ssh_dir "$name"`
   - Create dir with `mkdir -p`
   - If `id_claude` doesn't exist: generate with `ssh-keygen -t ed25519 -f "$ssh_dir/id_claude" -N "" -C "claude-sandbox-${name}"`
   - Set permissions: `chmod 600 "$ssh_dir/id_claude"`
   - If `ssh_host_ed25519_key` doesn't exist: generate host key similarly
   - Use msg_info/msg_ok for status output

2. `ssh_stage_build_keys(name)`:
   - Copy SSH keys from `common_ssh_dir "$name"` to `$CSM_ROOT/_build_ssh/`
   - Remove and recreate `_build_ssh/` dir to ensure clean state
   - Copy all files from ssh dir to _build_ssh/

3. `ssh_write_config(name, port)`:
   - Write SSH config block to `$HOME/.ssh/config`
   - Get alias via `common_ssh_alias "$name"`
   - Create `$HOME/.ssh` dir if needed (mode 700)
   - Remove any existing block for this alias (parse line by line, skip Host block matching alias)
   - Append new block:
     ```
     Host $alias
       HostName localhost
       Port $port
       User claude
       IdentityFile $ssh_dir/id_claude
       IdentitiesOnly yes
       StrictHostKeyChecking no
       UserKnownHostsFile /dev/null
     ```
   - Note: Linux doesn't have the Windows spaces-in-path issue, so IdentityFile points directly to the project ssh dir.

4. `ssh_remove_config(name)`:
   - Remove the SSH config block for this alias from `$HOME/.ssh/config`
   - Same parsing logic as write but only the removal step

Run `shellcheck lib/ssh.sh` and fix all issues.

  </verify>

  <files>lib/docker.sh, tests/docker.bats</files>
  <action>
1. Create `lib/docker.sh` with these functions:

   `docker_check_running()`:
   - Run `docker info &>/dev/null` or die "Docker is not running. Please start Docker and try again."

   `docker_build(name)`:
   - Build image tagged `claude-sandbox-$name`
   - Use: `docker build -t "claude-sandbox-${name}" -f "$CSM_ROOT/scripts/Dockerfile" "$CSM_ROOT"`
   - Check exit code, die on failure with msg_error

   `docker_run_instance(name, port)`:
   - Local vars for container_name (via common_container_name), workspace_dir (via common_workspace_dir)
   - Create workspace dir: `mkdir -p "$workspace_dir"`
   - Remove existing container first (BUG-01 fix): `docker rm -f "$container_name" 2>/dev/null || true`
   - Build the docker run command using a Bash array (per research anti-pattern advice):
     ```bash
     local cmd=(docker run -d)
     cmd+=(--name "$container_name")
     cmd+=(-p "127.0.0.1:${port}:22")           # SEC-02
     cmd+=(-v "${workspace_dir}:/home/claude/workspace")
     cmd+=(-w /home/claude/workspace)
     cmd+=(--memory=2g)                           # SEC-04
     cmd+=(--cpus=2)                              # SEC-04
     cmd+=(--security-opt=no-new-privileges)      # SEC-04
     cmd+=(--cap-drop=MKNOD)                      # SEC-03
     cmd+=(--cap-drop=AUDIT_WRITE)                # SEC-03
     cmd+=(--cap-drop=SETFCAP)                    # SEC-03
     cmd+=(--cap-drop=SETPCAP)                    # SEC-03
     cmd+=(--cap-drop=NET_BIND_SERVICE)           # SEC-03
     cmd+=(--cap-drop=SYS_CHROOT)                 # SEC-03
     cmd+=(--cap-drop=FSETID)                     # SEC-03
     cmd+=(--restart unless-stopped)
     cmd+=("claude-sandbox-${name}")
     "${cmd[@]}"
     ```
   - Check exit code, die on failure

   `docker_stop(name)`:
   - `docker stop "$(common_container_name "$name")" 2>/dev/null || true`

   `docker_remove(name)`:
   - `docker stop "$(common_container_name "$name")" 2>/dev/null || true`
   - `docker rm "$(common_container_name "$name")" 2>/dev/null || true`

   `docker_status(name)`:
   - `docker inspect --format '{{.State.Status}}' "$(common_container_name "$name")" 2>/dev/null || echo "not created"`

   `docker_start_instance(name)`:
   - Orchestrator function that calls the full flow:
     1. `ssh_ensure_keys "$name"`
     2. `ssh_stage_build_keys "$name"`
     3. Get or allocate port (check if already registered)
     4. `docker_build "$name"`
     5. `docker_run_instance "$name" "$port"`
     6. `ssh_write_config "$name" "$port"`
     7. msg_ok with instance name and port
     8. Return port for caller use
