  <name>Task 2: Add auto-backup hook to docker_start_instance and source backup.sh in bin/csm</name>
  <files>lib/docker.sh, bin/csm</files>
  <action>
**In lib/docker.sh -- `docker_start_instance` function:**

Add auto-backup check AFTER the `credentials_ensure_env_file` call and BEFORE the `docker_check_running` call. Insert this block:

```bash
# Auto-backup: capture last-known-good state before starting
credentials_load || true
if [[ "${CSM_AUTO_BACKUP:-}" == "1" ]]; then
    local status
    status="$(docker_status "$name")"
    if [[ "$status" == "running" || "$status" == "exited" ]]; then
        msg_info "Auto-backup: creating backup..."
        backup_create "$name"
    fi
fi
```

This checks CSM_AUTO_BACKUP from .env (parsed by credentials_load which reads ALL key=value pairs). If enabled AND the container exists (running or exited), it backs up before starting. If the container was never created, it silently skips (no container to back up).

**In bin/csm:**

Add `source "$CSM_ROOT/lib/backup.sh"` AFTER `source "$CSM_ROOT/lib/docker.sh"` and BEFORE `source "$CSM_ROOT/lib/menu.sh"`. Backup depends on docker.sh functions and is called by menu.sh functions, so this ordering is correct.

The source chain becomes:
```bash
source "$CSM_ROOT/lib/common.sh"
source "$CSM_ROOT/lib/credentials.sh"
source "$CSM_ROOT/lib/platform.sh"
source "$CSM_ROOT/lib/instances.sh"
source "$CSM_ROOT/lib/ssh.sh"
source "$CSM_ROOT/lib/docker.sh"
source "$CSM_ROOT/lib/backup.sh"    # NEW -- after docker.sh, before menu.sh
source "$CSM_ROOT/lib/menu.sh"
```
