<task type="auto">
  <name>Task 2: Extend entrypoint.sh with GUI variant detection and VNC startup</name>
  <files>scripts/entrypoint.sh</files>
  <action>
Extend the existing entrypoint.sh to detect the GUI variant and start VNC services before sshd. The existing credential injection code must remain unchanged at the top.

After the credential injection block (after the `fi` that closes the .bashrc grep), add GUI variant detection:

```bash
# Detect GUI variant by checking for vncserver binary
if command -v vncserver &>/dev/null; then
    # Set up VNC config for claude user (passwordless, localhost-only)
    mkdir -p /home/claude/.vnc
    cat > /home/claude/.vnc/config <<'VNCCONF'
SecurityTypes=None
geometry=1920x1080
depth=24
VNCCONF

    # xstartup: launch Xfce4 session
    cat > /home/claude/.vnc/xstartup <<'VNCSTART'
#!/bin/sh
unset SESSION_MANAGER
unset DBUS_SESSION_BUS_ADDRESS
exec startxfce4
VNCSTART
    chmod 755 /home/claude/.vnc/xstartup
    chown -R claude:claude /home/claude/.vnc

    # Start VNC server as claude user on display :1 (port 5901)
    su - claude -c "vncserver :1 -localhost yes" &

    # Wait for VNC server to be ready (poll, max 10 iterations)
    for i in $(seq 1 10); do
        if ss -tln 2>/dev/null | grep -q ':5901 ' || \
           netstat -tln 2>/dev/null | grep -q ':5901 '; then
            break
        fi
        sleep 0.5
    done

    # Start websockify: WebSocket-to-VNC bridge, serves noVNC UI on port 6080
    websockify --web=/usr/share/novnc/ --daemon 6080 localhost:5901
fi
```

The final line must remain `exec /usr/sbin/sshd -D` -- this is unchanged. The GUI services (vncserver, websockify) run as background processes/daemons; sshd runs in the foreground as PID 1.

Important notes:
- vncserver :1 runs as claude user via `su - claude -c` (not as root)
- Use a poll loop (not fixed sleep) for VNC readiness
- websockify runs with --daemon flag (backgrounds itself)
- SecurityTypes=None is safe because noVNC port is localhost-only bound on host side
