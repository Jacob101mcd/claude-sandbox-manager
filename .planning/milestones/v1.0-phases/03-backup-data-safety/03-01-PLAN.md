Create `lib/backup.sh` following the module pattern established in other lib/*.sh files:
- CSM_ROOT guard at top
- Module comment header listing provided functions

**Functions to implement:**

1. `backup_create "$name"`:
   - Get container_name via `common_container_name`
   - Check `docker_status` -- if "not created", `msg_warn` and return 1
   - Generate timestamp: `date +%Y%m%d-%H%M`
   - Create backup dir: `$(common_backup_dir "$name")/${timestamp}`
   - `docker commit "$container_name" "${container_name}-backup-${timestamp}"`
   - `docker save "$image_tag" | gzip > "${backup_dir}/image.tar.gz"`
   - `tar czf "${backup_dir}/workspace.tar.gz" -C "$(common_workspace_dir "$name")" .`
   - Write metadata.json via `jq -n` with --arg for imagetag, instance, port (from instances_get_port), type (from instances_get_type), timestamp
   - Report size via `du -sh "$backup_dir" | cut -f1`
   - `msg_ok "Backup complete: ${backup_dir} (${size})"`

2. `backup_list "$name"`:
   - Get backup_root via `common_backup_dir`
   - List subdirectories sorted reverse (newest first) using `find -mindepth 1 -maxdepth 1 -type d` piped through `sort -r`
   - If no dirs found, `msg_warn` and return 1
   - Print numbered list with size via `du -sh`
   - Store dirs in global `_BACKUP_LISTED_DIRS` array for caller use

3. `backup_restore "$name" "$backup_dir"`:
   - Read image_tag from `${backup_dir}/metadata.json` via `jq -r '.imagetag'`
   - Stop and remove container: `docker stop`, `docker rm` (with `2>/dev/null || true`)
   - Load image: `gunzip -c "${backup_dir}/image.tar.gz" | docker load`
   - Clear workspace contents: `rm -rf "${workspace_dir:?}"/*` (safe guard on empty var)
   - Extract workspace: `tar xzf "${backup_dir}/workspace.tar.gz" --no-same-owner -C "$workspace_dir"`
   - Get port from `instances_get_port "$name"`
   - Build docker run command array identical to `docker_run_instance` but with the backup image tag instead of `claude-sandbox-${name}`. Reuse the same security hardening flags (memory, cpus, cap-drops, no-new-privileges, restart policy, localhost SSH binding). Inject credentials via `credentials_load` + `credentials_get_docker_env_flags`.
   - `msg_ok "Restore complete. Instance '${name}' running from backup."`

**For the restore docker run:** Duplicate the docker run array from `docker_run_instance()` in docker.sh but parameterize the image tag. Do NOT call `docker_run_instance` directly because it uses `claude-sandbox-${name}` as the image tag. Copy the exact same flags. Add a code comment noting this must stay in sync with docker_run_instance.
