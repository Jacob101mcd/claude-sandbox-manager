    **2. Modify lib/docker.sh -- docker_run_instance():**
    After the existing security flags in the cmd array (before the image name line), add credential injection:
    ```bash
    # Load credentials and add -e flags
    credentials_load || true
    credentials_get_docker_env_flags
    cmd+=("${CSM_DOCKER_ENV_FLAGS[@]}")
    ```
    The `|| true` on credentials_load handles missing .env gracefully. credentials_get_docker_env_flags populates CSM_DOCKER_ENV_FLAGS and warns for missing keys.

    **3. Modify lib/docker.sh -- docker_start_instance():**
    Add a call to `credentials_ensure_env_file` at the beginning (before docker_check_running) so the .env template is auto-created on first use.

    **4. Modify lib/instances.sh -- instances_add():**
    Add a second parameter `$2` for container type (default "cli"):
    ```bash
    instances_add() {
        local name="$1"
        local type="${2:-cli}"
        ...
        jq --arg name "$name" --argjson port "$port" --arg type "$type" \
            '.[$name] = { "port": $port, "type": $type }' \
            ...
    }
    ```

    **5. Add instances_get_type() to lib/instances.sh:**
    ```bash
    instances_get_type() {
        local name="$1"
        _instances_ensure_file
        local type
        type="$(jq -r --arg name "$name" '.[$name].type // "cli"' "$_INSTANCES_FILE")"
        echo "$type"
    }
    ```
    Note the `// "cli"` default -- this handles backward compatibility with Phase 1 entries that lack a type field (Pitfall 5 from RESEARCH.md).

    **6. Modify lib/instances.sh -- instances_list_with_status():**
    Add type display. Change the msg_info line to include type:
    ```bash
    local type
    type="$(instances_get_type "$name")"
    msg_info "$name [$type] (port $port) - $status"
    ```
