<task type="auto">
  <name>Task 2: Overhaul Dockerfile with native installer and gh CLI</name>
  <files>scripts/Dockerfile</files>
  <action>
    Replace the existing scripts/Dockerfile with updated version. Key changes:

    1. Keep ubuntu:24.04 base image.

    2. System packages layer: Add `wget` to the existing apt-get install line (needed for gh CLI keyring download).

    3. Add GitHub CLI installation layer (after system packages, before Node.js):
       ```
       RUN wget -qO- https://cli.github.com/packages/githubcli-archive-keyring.gpg \
           | tee /etc/apt/keyrings/githubcli-archive-keyring.gpg > /dev/null \
           && chmod go+r /etc/apt/keyrings/githubcli-archive-keyring.gpg \
           && echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" \
           | tee /etc/apt/sources.list.d/github-cli.list > /dev/null \
           && apt-get update && apt-get install -y gh \
           && rm -rf /var/lib/apt/lists/*
       ```

    4. Keep Node.js LTS installation (needed for GSD framework).

    5. Keep user creation (claude user with passwordless sudo) -- same as current.

    6. Keep SSH setup -- same as current (key-only, no password).

    7. Keep SSH host key and authorized_keys COPY -- same as current.

    8. Replace NPM Claude Code install with native installer (as claude user):
       ```
       USER claude
       RUN curl -fsSL https://claude.ai/install.sh | bash
       RUN echo 'export PATH="$HOME/.local/bin:$PATH"' >> /home/claude/.bashrc \
           && echo 'export PATH="$HOME/.local/bin:$PATH"' >> /home/claude/.profile
       ```
       Remove the old `RUN npm install -g @anthropic-ai/claude-code` line.

    9. Keep GSD framework install (as claude user, after native installer).

    10. Keep EXPOSE 22 and CMD ["/usr/sbin/sshd", "-D"].

    Layer ordering rationale: system packages (rarely change) -> gh CLI (rarely changes) -> Node.js (rarely changes) -> user setup (rarely changes) -> SSH (rarely changes) -> native installer (may update) -> GSD (may update). This maximizes Docker cache hits.

    CRITICAL: Do NOT add any ARG or ENV instructions for credentials. Credentials are injected at runtime only (CRED-04).
  </action>
