  <name>Task 1: Create lib/menu.sh interactive menu and bin/csm entry point</name>
  <files>lib/menu.sh, bin/csm</files>
  <action>
1. Create `lib/menu.sh` implementing the interactive loop, modeled after scripts/claude-manager.ps1:

   `menu_show_header()`:
   - Print colored banner: "=== Claude Sandbox Manager ===" in green
   - Print blank line

   `menu_show_instances()`:
   - Call `instances_list_with_status` to display all instances with their Docker status
   - If no instances, print "(no instances registered)" in gray

   `menu_show_actions()`:
   - Print "--- Actions ---" header in cyan
   - Print: [S] Start an instance
   - Print: [T] Stop an instance
   - Print: [N] Create new instance
   - Print: [R] Remove an instance
   - Print: [Q] Quit

   `menu_select_instance(prompt)`:
   - Get all instances from registry
   - If zero: msg_warn "No instances found." and return 1
   - If one: auto-select and print "Using instance: $name"
   - If multiple: show numbered list, read choice, validate index
   - Echo selected name to stdout (caller captures with `$(menu_select_instance "...")`)

   `menu_action_start()`:
   - Call menu_select_instance "Select instance to start:"
   - If selected: call `docker_start_instance "$name"`
   - After successful start, prompt: "SSH into instance now? (y/N)"
   - If "y" or "Y": exec `ssh "$(common_ssh_alias "$name")"`

   `menu_action_stop()`:
   - Call menu_select_instance "Select instance to stop:"
   - If selected: call `docker_stop "$name"`, msg_ok "Stopped."

   `menu_action_new()`:
   - Read instance name from user: "Enter instance name (lowercase, no spaces):"
   - Sanitize: `name=$(echo "$input" | tr -cd 'a-z0-9-')`
   - Validate: non-empty, not already in registry
   - Call `docker_start_instance "$name"`
   - After start, prompt SSH (same as menu_action_start)

   `menu_action_remove()`:
   - Call menu_select_instance "Select instance to remove:"
   - Confirm: "Remove instance '$name'? This stops the container and deregisters it. Type YES to confirm"
   - If confirmed:
     - `docker_remove "$name"`
     - `ssh_remove_config "$name"`
     - `instances_remove "$name"`
     - Ask about deleting files: "Also delete workspace and backups for '$name'? (y/N)"
     - If yes: remove workspace dir, backup dir, ssh dir

   `menu_main()`:
   - Auto-create "default" instance if no instances exist (like PowerShell version)
   - Infinite loop:
     - Show instances
     - Show actions
     - Read choice
     - Dispatch to action function based on choice (case statement, case-insensitive)
     - Invalid choice: msg_error "Invalid choice."

2. Create `bin/csm`:
   - `#!/usr/bin/env bash`
   - `set -euo pipefail`
   - Resolve CSM_ROOT: `CSM_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"`
   - Export CSM_ROOT
   - Source all lib modules in dependency order:
     ```bash
     source "$CSM_ROOT/lib/common.sh"
     source "$CSM_ROOT/lib/platform.sh"
     source "$CSM_ROOT/lib/instances.sh"
     source "$CSM_ROOT/lib/ssh.sh"
     source "$CSM_ROOT/lib/docker.sh"
     source "$CSM_ROOT/lib/menu.sh"
     ```
   - Call `platform_detect`
   - Call `platform_check_dependencies`
   - Call `docker_check_running`
   - Call `menu_main`
   - `chmod +x bin/csm`

3. Run `shellcheck bin/csm lib/menu.sh` and fix all issues.

4. Run the full ShellCheck suite: `shellcheck bin/csm lib/*.sh`

5. Run the full BATS suite: `bats tests/`

6. Verify Windows scripts are UNCHANGED: `git diff scripts/*.ps1` should show no changes.
