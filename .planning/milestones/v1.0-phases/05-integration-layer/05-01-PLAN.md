<task type="auto" tdd="true">
  <name>Task 2: Linux Engine detection, --add-host flag, and remote control menu prompt</name>
  <files>lib/docker.sh, lib/menu.sh, tests/docker.bats, tests/menu.bats</files>
  <behavior>
    - docker_run_instance on Linux Engine (not Desktop) adds --add-host=host.docker.internal:host-gateway
    - docker_run_instance on macOS or Docker Desktop Linux does NOT add --add-host
    - menu_action_new prompts "Enable remote control?" after container type selection
    - Default answer is N (remote_control stays false)
    - Answering y/Y sets remote_control to true in instances.json
    - After start, if remote_control is true, print info message about remote control
  </behavior>
  <action>
    1. In lib/docker.sh — add Linux Engine detection and --add-host flag:
       - Add a helper function `_docker_detect_variant()` that returns "desktop" or "engine":
         ```
         local ctx
         ctx="$(docker context show 2>/dev/null || echo "default")"
         if [[ "$ctx" == "desktop-linux" ]]; then echo "desktop"; return; fi
         if docker context inspect desktop-linux &>/dev/null 2>&1; then echo "desktop"; return; fi
         echo "engine"
         ```
       - In docker_run_instance, after the GUI-specific flags block and before the credentials block, add:
         ```
         # MCP: ensure host.docker.internal resolves on Linux Engine
         if [[ "$(uname -s)" == "Linux" ]]; then
             local docker_variant
             docker_variant="$(_docker_detect_variant)"
             if [[ "$docker_variant" == "engine" ]]; then
                 cmd+=(--add-host=host.docker.internal:host-gateway)
             fi
         fi
         ```

    2. In lib/menu.sh — add remote control prompt to menu_action_new:
       - After `instances_add "$name" "$container_type"` line, add:
         ```
         # Prompt for remote control (default off per locked decision)
         local rc_answer
         read -rp "Enable remote control (requires claude.ai account, not API key)? (y/N) " rc_answer
         if [[ "$rc_answer" == "y" || "$rc_answer" == "Y" ]]; then
             instances_set_remote_control "$name" true
         fi
         ```
       - After docker_start_instance completes and before the SSH/browser prompt, add a check:
         ```
         local rc_enabled
         rc_enabled="$(instances_get_remote_control "$name")"
         if [[ "$rc_enabled" == "true" ]]; then
             msg_info "Remote control enabled. After SSH, check /tmp/csm-remote-control.log for session URL."
         fi
         ```
